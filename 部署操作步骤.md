# 🚀 部署操作步骤（详细版）

## ⚠️ 重要说明

**我无法直接访问您的 GitHub 账号**，但我会帮您准备好所有文件，您只需要在浏览器中操作即可。

**不需要密码**，只需要：
1. 在浏览器中登录 GitHub（OAuth 授权）
2. 按照步骤操作

---

## 📋 第一步：创建 GitHub 仓库

### 方法一：通过网页创建（推荐）

1. **打开 GitHub**
   - 访问：https://github.com
   - 登录您的账号（如果没有，先注册）

2. **创建新仓库**
   - 点击右上角 **+** → **New repository**
   - 仓库名称：`auction-game`（或您喜欢的名字）
   - 描述：`臻藏拍卖 - 现代化艺术品竞拍平台`
   - 选择：**Public**（公开，免费使用 GitHub Pages）
   - **不要**勾选 "Initialize this repository with a README"
   - 点击 **Create repository**

3. **复制仓库地址**
   - 创建后会显示仓库地址，类似：
   ```
   https://github.com/你的用户名/auction-game.git
   ```
   - **复制这个地址**，下一步需要用到

---

## 📋 第二步：连接本地代码到 GitHub

我已经帮您准备好了 Git 仓库，现在只需要连接：

### 在终端中执行（我已经准备好了命令）

```bash
# 替换下面的 URL 为您的实际仓库地址
git remote add origin https://github.com/你的用户名/auction-game.git
git branch -M main
git push -u origin main
```

**注意**：执行 `git push` 时，浏览器会自动打开，要求您登录 GitHub 并授权。

---

## 📋 第三步：启用 GitHub Pages

1. **打开仓库页面**
   - 访问：`https://github.com/你的用户名/auction-game`

2. **进入设置**
   - 点击顶部 **Settings** 标签

3. **配置 Pages**
   - 左侧菜单找到 **Pages**
   - **Source** 选择：
     - Branch: `main`
     - Folder: `/ (root)`
   - 点击 **Save**

4. **等待部署**
   - 等待 1-2 分钟
   - 刷新页面，会显示部署地址：
   ```
   https://你的用户名.github.io/auction-game/
   ```

---

## 🎉 完成！

部署完成后，您的网站就可以通过以下地址访问：
```
https://你的用户名.github.io/auction-game/
```

## 🆘 遇到问题？

### Q: git push 时要求输入用户名密码？
**A**: 使用 Personal Access Token：
1. GitHub → Settings → Developer settings → Personal access tokens → Tokens (classic)
2. Generate new token → 勾选 `repo` 权限
3. 复制 token，作为密码使用

### Q: 找不到 Settings → Pages？
**A**: 确保仓库是 **Public**（公开），私有仓库需要付费才能使用 Pages。

### Q: 部署后页面空白？
**A**: 
1. 检查浏览器控制台（F12）是否有错误
2. 确保所有文件都已上传（检查 GitHub 仓库文件列表）
3. 等待几分钟后刷新
