## 🚀 部署步骤（5 分钟）

### 第 1 步：创建 GitHub 仓库（2 分钟）

1. **打开浏览器，访问**：https://github.com
2. **登录您的 GitHub 账号**（如果没有，先注册，免费）
3. **点击右上角 + 号** → **New repository**
4. **填写信息**：
   - Repository name: `auction-game`
   - Description: `臻藏拍卖 - 现代化艺术品竞拍平台`
   - 选择：**Public** ✅
   - **不要勾选** "Add a README file"
   - 点击 **Create repository**

5. **复制仓库地址**（创建后会显示）：
   ```
   https://github.com/你的用户名/auction-game.git
   ```

---

### 第 2 步：推送代码到 GitHub（1 分钟）

**在终端中执行以下命令**（替换 `你的用户名` 为实际的 GitHub 用户名）：

```bash
git remote add origin https://github.com/你的用户名/auction-game.git
git branch -M main
git push -u origin main
```

**执行 `git push` 时**：
- 浏览器会自动打开
- 要求您登录 GitHub（OAuth，不需要密码）
- 授权后代码会自动上传

---

### 第 3 步：启用 GitHub Pages（2 分钟）

1. **打开仓库页面**：`https://github.com/你的用户名/auction-game`
2. **点击顶部 "Settings" 标签**
3. **左侧菜单找到 "Pages"**
4. **配置**：
   - Source: 选择 **main** 分支
   - Folder: 选择 **/ (root)**
   - 点击 **Save**
5. **等待 1-2 分钟**
6. **刷新页面**，会显示您的网站地址：
   ```
   https://你的用户名.github.io/auction-game/
   ```

## 🆘 遇到问题？

### 问题 1：git push 失败
**解决方案**：
- 确保已创建 GitHub 仓库
- 检查仓库地址是否正确
- 如果要求输入密码，使用 Personal Access Token（见下方）

### 问题 2：需要输入密码
**解决方案**：
1. GitHub → Settings → Developer settings
2. Personal access tokens → Tokens (classic)
3. Generate new token
4. 勾选 `repo` 权限
5. 复制 token，作为密码使用

### 问题 3：找不到 Pages 设置
**解决方案**：
- 确保仓库是 **Public**（公开）
- 私有仓库需要付费才能使用 Pages
